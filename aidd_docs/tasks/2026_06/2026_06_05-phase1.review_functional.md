## Flow / Edge-case Gaps

Gaps relevés en parcourant chaque critère contre le diff.

- [ ] **HOME-AC7 — `catch on Exception` ne couvre pas `StateError` (Major).** `accueil_bloc.dart:68` capture `on Exception` ; or `conseilDuJour` lève `StateError` (l.126 `app_database.dart`), qui est un `Error` et **pas** un `Exception`. Si la base est vide (seed échoué), l'exception n'est pas attrapée → pas de fallback État A → risque de crash. Le contrat AC7 (« ne jamais crasher l'écran ») n'est donc pas garanti sur ce chemin. Suggéré : `on Object` (comme dans `DemarrageBloc`) ou capter `StateError`.
- [ ] **FND-AC4 — edge minuit `observerDerniereHumeurDuJour` (Minor).** `isBetweenValues(start, end)` est inclusif sur `end` (minuit+1j). La requête `LIMIT 1 DESC` pourrait sélectionner une entrée horodatée exactement à `end` ; le `.map` la transforme alors en `null` (l.112-113), **masquant** une éventuelle entrée légitime plus tôt dans la journée. Cas extrêmement rare (timestamp = minuit pile du lendemain) mais le filtrage post-requête peut renvoyer `null` à tort. Suggéré : exclure `end` dans la clause `where` (`creeLe < end`) plutôt qu'en post-filtrage.
- [ ] **HOME-AC5 — `HapticFeedback` non confirmé (Major).** Le plan §6 exige `HapticFeedback.lightImpact()` sur chaque geste. Aucun appel `HapticFeedback` visible dans `accueil_view.dart` ni `carte_humeur.dart` (les gestes appellent directement `ouvrirPlaceholder`). À vérifier : soit `ouvrirPlaceholder` (dans `common/placeholder_screen.dart`) déclenche le haptique, soit le critère est partiellement non tenu. Fichier `placeholder_screen.dart` non confirmé sur ce point dans cette revue.
- [ ] **S-AC8 / FND-AC8 / S-AC5 — splash natif non vérifiable en statique (Minor).** La génération réelle des drawables Android / `LaunchScreen` iOS et l'absence de flash blanc nécessitent un build (`dart run flutter_native_splash:create`) + exécution sur device. La config est correcte ; la **preuve d'exécution** manque (hors périmètre revue statique). À valider en build manuel + sur API 31+ (Android 12).
- [ ] **Incohérence codes émotion 6 vs 7 (signalée dans les plans, Minor).** Le corps des plans Accueil listait 6 libellés (`neutral/anxious`) ; le FONT FOI fige **7** (`happy/calm/dynamic/sad/angry/nervous/tired`). L'implémentation suit bien le FONT FOI (`emojiParCode`, `_libelleEmotion`, ARB `moodDynamic/moodNervous/moodTired`). Cohérent — l'arbitrage est résolu côté code dans le sens FONT FOI. Pas un défaut, confirmation de bonne résolution.
- [ ] **Reduced motion Démarrage — couverture des widgets halo/anneaux (Minor).** `demarrage_view.dart` passe `animer` à `HaloRespirant`/`AnneauxOndes`/`PointsChargement` ; `points_chargement.dart` honore bien `animer=false`. Vérifier que `anneaux_ondes.dart` et `demarrage/widgets/halo_respirant.dart` rendent aussi un état statique quand `animer=false` (non relus intégralement ici — le pattern est cohérent mais à confirmer pour 100% de couverture S-AC6).
- [ ] **`AccueilDemarre` non re-déclenché au retour de placeholder.** Conforme au plan (le stream Drift pilote A/B sans refresh manuel) — pas un gap, mais dépend de la persistance de l'abonnement `emit.forEach` tant que la page Accueil n'est pas détruite. OK car `pushReplacement` n'est pas utilisé pour les placeholders (push standard), donc Accueil reste monté.
